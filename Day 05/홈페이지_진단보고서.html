현재 홈페이지는 브랜드 분위기(감성, 폰트, 애니메이션)는 잘 잡혀 있으나, PRD가 정의한 핵심 가치 "바쁜 아침에 대기 없이 픽업"을 전달하지 못하고 있다. 픽업 예약이 페이지 맨 하단에 위치하고, Hero 카피는 방문 동기를 유발하지 않으며, 예약 완료 후 피드백이 없어 사용자 신뢰를 떨어뜨린다.

진단 항목

🔴

Issue 1 · 긴급

Hero 카피가 "빠른 픽업" 가치를 전달하지 않음

위치 Hero 섹션 line 542–546
현재 문구 "매일 아침 직접 내리는 핸드드립, 당일 구운 수제 스콘과 함께 하루를 시작하는 곳"
문제 무엇을 주는지(What)만 설명, 왜 지금 선택해야 하는지(Why)가 없음. 출근 전 직장인 페르소나가 스타벅스 대신 선택할 이유가 보이지 않음
PRD 근거 PRD §1 — "편의성 부재로 선택에서 탈락" / 페르소나 핵심 니즈: "대기 없이 픽업"
개선 방향 ✓ "미리 주문, 도착하면 바로 픽업" — 시간 절약 메시지로 전환
🔴

Issue 2 · 긴급

픽업 예약 버튼이 3번째 낮은 우선순위

위치 Hero 버튼 그룹 line 548–551
현재 순서 ① 메뉴 보기 (fill, 강조색) → ② 오시는 길 → ③ 픽업 예약 (ghost, 3번째)
문제 PRD 핵심 전환 행동인 픽업 예약이 비강조·3순위 배치. 방문자 시선이 닿기 전 이탈 가능
PRD 근거 PRD §4 — 카카오채널 픽업 예약 Must Have / 플로우 A가 핵심 플로우
개선 방향 ✓ 픽업 예약 → 1번째 fill 버튼으로 격상, 메뉴 보기 → ghost 2순위
🔴

Issue 3 · 긴급

예약 완료 후 사용자 피드백 없음

위치 addReservation() 함수 line 768–792
현재 동작 예약 추가 → 폼 초기화 → inpName.focus() — 아무 메시지 없이 종료
문제 사용자가 예약이 등록됐는지 불확실. 모바일에서 목록이 하단에 있어 확인 어려움
개선 방향 ✓ 하단 토스트 메시지 "✅ OOO님 픽업 예약이 완료됐습니다" 2.5초 표시
🟡

Issue 4 · 중간

"오늘의 원두" 정적 텍스트

위치 메뉴 섹션 두 번째 카드 line 573–577
현재 문구 "매일 다른 산지의 원두로 만나는 오늘" (고정 텍스트)
문제 방문자가 오늘 실제 원두를 알 수 없음. PRD 사용자 목표 "30초 내 오늘 메뉴 확인" 미달
PRD 근거 PRD §2 — "방문 전 오늘의 메뉴·재고를 30초 안에 확인할 수 있다"
개선 방향 JS 변수로 관리자 수정 가능하게 구현 (단기) / 인스타 스토리 링크 배치 (즉시)
🟡

Issue 5 · 중간

카카오채널 픽업 예약 연결 없음

위치 CTA 섹션 line 629–651, 예약 섹션 전체
현재 상태 Instagram DM + 전화번호만 안내. 카카오채널 언급 없음
문제 PRD Must Have인 카카오채널 픽업 예약이 홈페이지에 없음. 현재 예약 폼은 내부 관리용
개선 방향 ✓ CTA 섹션에 카카오채널 버튼 추가 (pf.kakao.com 링크)
🟢

Issue 6 · 낮음

실제 주소 없음 — 지도 연결 불가

위치 Info 섹션 Location 블록 line 618–621
현재 문구 "서울 망원동 골목 끝 / 조용한 골목 안 작은 카페"
문제 실제 도로명 주소 없음 → 네이버·카카오 지도 링크 연결 불가. 신규 방문자가 찾아갈 수 없음
개선 방향 실제 주소 입력 + 네이버 지도 링크 버튼 추가
🟢

Issue 7 · 낮음

영업 오픈 시간이 PRD 플로우와 불일치

위치 Info 섹션 Hours line 603, Footer line 721
현재 표기 "오전 08:00 — 오후 20:00"
문제 PRD 플로우 A는 "07:20 인스타 확인 → 08:05 픽업" 시나리오. 07시대 픽업 수용 가능 여부 불명확
개선 방향 실제 오픈 시간 오프라인 확인 후 반영 (PRD Open Question 해소 선행)
