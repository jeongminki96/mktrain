# 모닝브루 홈페이지 진단 보고서
**작성일:** 2026-06-20  
**진단 대상:** `index.html` (현재 루트 배포본)  
**비교 기준:** `PRD_lite_v0.1.md` (Day 02)

## 요약 (Executive Summary)

현재 홈페이지는 브랜드 분위기(감성, 폰트, 애니메이션)는 잘 잡혀 있으나, PRD가 정의한 **핵심 가치("바쁜 아침에 대기 없이 픽업")를 전달하지 못하고** 있다. 픽업 예약이 페이지 맨 하단에 위치하고, Hero 카피는 방문 동기를 유발하지 않으며, 예약 완료 후 피드백이 없어 사용자 신뢰를 떨어뜨린다.

**긴급도 높음 3건 / 중간 2건 / 낮음 2건**

## 진단 항목

### 🔴 Issue 1 — Hero 카피가 "빠른 픽업" 가치를 전달하지 않음

| 항목 | 내용 |
|------|------|
| 위치 | `index.html` Hero 섹션 (line 542–546) |
| 현재 문구 | "매일 아침 직접 내리는 핸드드립, 당일 구운 수제 스콘과 함께 하루를 시작하는 곳" |
| 문제 | **무엇을 주는지(What)**만 설명, **왜 지금 선택해야 하는지(Why)**가 없음. 출근 전 직장인 페르소나가 스타벅스 대신 여기를 선택할 이유가 보이지 않음 |
| PRD 근거 | PRD §1: "편의성 부재로 선택에서 탈락" / 페르소나: "대기 없이 픽업" 핵심 니즈 |
| 개선 방향 | "주문 미리, 픽업만" / "3분 픽업, 하루를 여는 한 잔" 등 **시간 절약** 메시지로 전환 |

---

### 🔴 Issue 2 — 픽업 예약 버튼이 3번째 낮은 우선순위

| 항목 | 내용 |
|------|------|
| 위치 | Hero 버튼 그룹 (line 548–551) |
| 현재 순서 | ① **메뉴 보기** (fill, 강조색) → ② 오시는 길 (ghost) → ③ 픽업 예약 (ghost) |
| 문제 | PRD 핵심 전환 행동인 픽업 예약이 3번째·비강조 버튼으로 배치. 방문자의 시선이 닿기 전에 스크롤하거나 이탈 |
| PRD 근거 | PRD §4: 카카오채널 픽업 예약 Must Have / 플로우 A가 핵심 플로우 |
| 개선 방향 | **픽업 예약 → 1번째 fill 버튼**으로 격상, 메뉴 보기는 2번째 ghost로 전환 |

### 🔴 Issue 3 — 예약 완료 후 피드백 없음

| 항목 | 내용 |
|------|------|
| 위치 | `addReservation()` 함수 (line 768–792), `clearForm()` (line 846–849) |
| 현재 동작 | 예약 추가 → 폼 초기화 → `inpName.focus()` — 아무 메시지 없이 끝남 |
| 문제 | 사용자가 예약이 등록됐는지 불확실. 특히 모바일에서 목록이 하단에 있어 확인이 어려움 |
| 개선 방향 | 토스트 메시지("✅ 예약이 완료됐습니다") 또는 버튼 텍스트 1.5초 변경("완료!") |
