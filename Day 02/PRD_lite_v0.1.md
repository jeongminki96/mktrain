## 1. 배경 & 문제 정의

### 현재 상황 (As-Is)

서울 마포구 망원동 골목 끝에 위치한 소규모 스페셜티 카페. 핸드드립 중심, 당일 구운 스콘, 오늘의 원두 큐레이션이 강점이지만, 주요 수익원인 **출근 전 직장인 고객**을 체계적으로 붙잡는 운영 장치가 없다.

### 문제 (Problem Statement)

> **"맛은 알지만, 바쁜 아침에 선택하기 어렵다."**

출근 전 시간이 촉박한 고객은 ① 오늘 메뉴를 사전에 알 수 없고 ② 대기가 길면 이탈해 스타벅스·편의점으로 향한다. 모닝브루는 품질로 이길 수 있지만, **편의성 부재**로 선택에서 탈락되고 있다.

### 근거

| 출처 | 핵심 내용 |
|------|---------|
| 시장조사_타깃페인 #1 | 피크타임(08~09시) 대기 중 이탈, 메가커피·편의점으로 대체 |
| 시장조사_타깃페인 #2 | 오늘의 원두·스콘 재고를 방문 전 확인 불가 |
| 시장조사_타깃페인 #3 | 사전 픽업 주문 불가 → 사이렌오더 되는 스타벅스로 이탈 |
| 경쟁사_비교분석 | 망원동 내 핸드드립+편의성 동시 제공 카페 없음 → 공백 존재 |

## 2. 목표 (Goals)

### 비즈니스 목표
- [ ] 출근 전 피크타임(07~09시) 객단가·회전율 유지하면서 이탈 고객 수 감소
- [ ] 단골 재방문율 향상 (현재 측정값 없음 → 기준선 설정이 선행 과제)
- [ ] 원두 소매 판매로 부가 수익원 1개 이상 확보

### 사용자 목표
- [ ] 방문 전 오늘의 메뉴·재고를 30초 안에 확인할 수 있다
- [ ] 출근 동선에서 대기 없이 커피를 픽업할 수 있다
- [ ] "여기 오면 내 취향을 알아준다"는 단골 경험을 갖는다

### 비목표 (Non-Goals) — 이번 범위 제외
- 키오스크·POS 시스템 도입 (투자 규모 초과)
- 배달앱 입점 (소규모 운영 부담, 수수료 구조 불리)
- 2호점 확장
- 대규모 SNS 광고 집행

## 4. 핵심 기능 요구사항

> 우선순위: 🔴 Must Have / 🟡 Should Have / 🟢 Nice to Have

| 우선순위 | 기능 | 사용자 스토리 | 완료 조건 |
|---------|------|------------|---------|
| 🔴 | **오늘의 원두·메뉴 인스타 스토리 정례 업데이트** | 나는 출근 전 고객으로서, 오늘 어떤 원두인지 미리 알고 방문 여부를 결정하고 싶다 | 평일 매일 07:30 이전 인스타 스토리에 당일 원두명·산지·스콘 재고 공지 |
| 🔴 | **카카오채널 픽업 예약 (간이 사전 주문)** | 나는 시간이 없는 직장인으로서, 도착 전 미리 주문하고 기다림 없이 픽업하고 싶다 | 카카오 채널 채팅으로 메뉴·픽업 시간 접수, 완료 답장 3분 내 발송 |
| 🔴 | **결제 수단 전면 확대** | 나는 방문 고객으로서, 카드·카카오페이·네이버페이 등 모든 간편결제로 편하게 내고 싶다 | 카드·카카오페이·네이버페이·삼성페이 전체 가능 여부 확인 및 안내 게시 |

| 🟡 | **원두 소매 판매 (오프라인 → 온라인 확장)** | 나는 모닝브루 단골로서, 집에서도 같은 원두로 커피를 내려 마시고 싶다 | 매장 내 원두 소분 판매 + 네이버 스마트스토어 시범 등록 1종 |
| 🟢 | **단골 취향 카드 (아날로그 CRM)** | 나는 자주 오는 단골로서, 바리스타가 내 취향을 기억해주는 경험을 원한다 | 단골 10명 대상 수기 취향 카드 운영, 재방문 시 맞춤 원두 제안 |
| 🟢 | **주말 좌석 현황 인스타 스토리 공지** | 나는 주말 방문 고객으로서, 자리 있는지 확인 후 나서고 싶다 | 주말 10시·12시 2회 좌석 현황 스토리 업로드 |

## 5. 사용자 흐름 (User Flow)

### 플로우 A — 출근 전 픽업 (핵심 플로우)

```
[오전 07:20] 인스타 스토리 확인
     → 오늘의 원두·스콘 재고 확인
     → 카카오채널로 픽업 주문 전송
     → 모닝브루 확인 답장 수신
     → [08:05] 매장 도착 → 바로 픽업
     → 대기 없이 출근길 합류
```

### 플로우 B — 주말 방문 (보조 플로우)

```
[오전 09:50] 인스타 스토리 "자리 여유 있음" 확인
     → 도보 이동
     → 오늘의 원두 설명 듣고 주문
     → 원두 소매 구매 고려
```

## 7. 제약 & 리스크

| 유형 | 내용 | 대응 방안 |
|------|------|---------|
| 운영 제약 | 1인 운영 가능성 → 인스타 업로드·채널 응대 부담 | 전날 밤 예약 게시, 카카오채널 자동응답 문구 설정 |
| 기술 제약 | 카카오채널 픽업 예약은 수기 접수 방식 → 동시 다발 주문 처리 한계 | 초기 하루 최대 5건으로 제한, 수요 확인 후 확대 |
| 품질 리스크 | 사전 주문 폭주 시 핸드드립 품질 유지 어려움 | 픽업 메뉴를 에스프레소 계열 또는 배치 브루로 제한 옵션 검토 |
| 측정 리스크 | 현재 기준선(baseline) 데이터 없음 | 개선 전 2주간 판매 수기 기록으로 기준선 먼저 확보 |
