Google Crawler — 기술적·기계적 파싱 관점

✅ 잘 된 점

  • JSON-LD CafeOrCoffeeShop 스키마가 Menu·MenuItem·OpeningHoursSpecification·GeoCoordinates까지 포함 — 구조화 데이터 완성도 높음
  • FAQPage를 Microdata 방식으로 이중 구현 → Rich Result 트리거 경로 두 개 확보

❌ 문제점

  • 예약 폼이 JS 동적 콘텐츠 → 초기 HTML 파싱 시 실질 콘텐츠로 미인식

Verdict

"JSON-LD와 FAQ 구조화 데이터 구현 수준은 높으나, 이미지 전무·NAP 허구값·og:image 누락이 로컬 SEO 및 이미지 검색 신호를 완전히 차단하여 실전 순위 경쟁력이 낮음."

생성형 AI 엔진

ChatGPT / Perplexity — 정보 추출·인용 가능성 관점

74
B

✅ 답변 가능한 질문

  • "모닝브루 영업시간이 어떻게 돼요?" → JSON-LD + FAQPage + 본문 삼중 일치, 고신뢰 답변
  • "픽업 예약 방법 알려줘" → FAQPage acceptedAnswer 직접 인용 가능
  • "망원동 핸드드립 커피 가격" → MenuItem + Offer 4종 파싱, 표 형태 답변 생성 가능

❌ 답변 불가한 질문

  • "정확한 주소 알려줘" → streetAddress 자연어 서술, 도로명·우편번호 없음
  • "주차 가능한가요?" → amenityFeature 정보 전무
  • "오늘 자리 있나요?" → 좌석 수·혼잡도 정보 없음
  • "와이파이 돼요?" → 편의시설 정보 전무

⚠️ 신뢰도 문제

  • 전화번호 02-123-4567 가상 번호 → AI 인용 시 사용자 연락 실패
  • JSON-LD에 @id 없음 → Knowledge Graph 연결 약화
  • SNS URL이 검증되지 않은 가상 링크 → Perplexity가 404 확인 시 신뢰 페널티

Verdict

"JSON-LD + FAQPage 이중 구조화로 영업시간·픽업 방법·가격 답변에는 충분히 활용 가능하나, 실 도로명 주소와 실제 연락처가 없어 AI가 '신뢰할 수 있는 출처'로 인용하기에는 치명적 허점이 남아 있다."

🚶 검색 여정 단계별 경험

검색 단계

title + description에서 "미리 주문하고 바로 픽업" 확인 → 클릭 의지 생김. OG 태그도 잘 세팅되어 SNS 미리보기도 괜찮음.

클릭 단계

Hero에 픽업 메시지는 바로 보이지만, 영업시간이 첫 화면에 없어 "지금 열었나?" 5초 안에 확인 불가. 스크롤 필요.

페이지 내 단계

예약 폼이 4번째 섹션 맨 하단. 폼 입력 후 카페에 실제 전달됐는지 불분명 (로컬스토리지에만 저장). 지도 링크 없음.

✅ 좋았던 점

  • "미리 주문, 도착하면 바로 픽업" 문구가 Hero 정중앙 — 내 상황과 딱 맞는 카페임을 첫눈에 알았다
  • 영업시간이 스크롤하면 바로 명확하게 나온다
  • FAQ에 "오늘의 원두 어디서 확인?" 답변이 있어 스콘 허탕 막는 법을 알 수 있었다

❌ 불편했던 점

  • Hero에 영업시간 없음 — "지금 열었나?" 확인을 두 번 스크롤해야 한다
  • 픽업 예약 폼이 맨 하단 — 메뉴·오시는 길·CTA 세 섹션을 지나야 도달
  • 예약 후 카페 측 전달 여부 불명확 — 로컬스토리지에만 저장, 확인 피드백 없음
  • "오시는 길"에 지도 링크 없음 — 처음 가는 골목 카페, 지도 앱 바로 열려야 한다
  • 스콘 재고 현황 실시간 확인 불가 — "재고 있음/소진" 배지 하나만 있어도 됐다

Verdict (박지수 목소리)

"픽업 된다고 해서 눌렀는데, 폼이 맨 아래 있고 예약이 카페에 실제로 전달되는지도 모르겠어서 결국 카카오 채널로 따로 문자 보냈다 — Hero에 영업 중 배지랑 지도 링크만 있었어도 진작 예약했을 텐데."

  • 실 도로명 주소 없음 → NAP 일관성 0, 로컬 팩 진입 불가
  • 본문 텍스트 약 600자 → 경쟁 페이지(1,500~3,000자) 대비 키워드 관련성 열위

  • 네이버 플레이스 sameAs 없음 → 국내 검색 점유율 1위 채널 단절
  • sitemap.xml lastmod 미래 날짜 → GSC 색인 우선순위 불이익

🎯 당장 1순위 액션

  • JSON-LD streetAddress에 실제 도로명 주소 + postalCode 추가 → 네이버 플레이스 등록 → sameAs 추가 (로컬 SEO ROI 최고)

Verdict

"기술 구현 기반은 탄탄하지만, 실 주소 누락·og:image 없음·본문 텍스트 빈약이라는 로컬 SEO 3대 치명상이 동시에 존재하여 '망원동 카페' 지역 검색에서 경쟁 카페를 이기려면 아직 갈 길이 멀다."

📡 네이버 최적화 신호

  • ❌ 주소 불완전: "망원동 골목 끝" → 플레이스 색인 불가, postalCode 없음
  • ❌ 가상 전화번호: 네이버 플레이스 업체 검증 실패
  • ❌ 네이버 플레이스 URL sameAs 없음: 지역 검색 핵심 경로 단절

💛 카카오 생태계 신호

  • ✅ 카카오채널 연동: sameAs + CTA 버튼 존재 → 카카오맵 신뢰 신호 기여 가능
  • ⚠️ 카카오맵 임베드 없음: "오시는 길" 섹션 텍스트 전용, 지도 임베드 미존재

❌ 핵심 문제점

  • 네이버 플레이스 미등록 구조 — 카페 검색 80% 이상이 이 경로, 홈페이지만 있으면 지역 검색 노출 제로
  • 실 주소 부재 → 네이버 지도·카카오맵 핀 생성 불가
  • VIEW 탭(블로그·카페 리뷰) 유입 콘텐츠 없음
  • og:image 없음 → 네이버 카페·블로그 공유 시 썸네일 없음
  • canonical이 GitHub Pages URL → 실제 도메인 없이 신뢰도 낮음

Verdict

"한국어 키워드와 구조화 데이터는 준비되어 있으나, 실 주소·실 전화번호 부재와 네이버 플레이스 미연동으로 인해 한국 사용자가 카페를 찾는 핵심 경로에서 완전히 비가시적인 상태."
