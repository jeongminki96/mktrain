Section 01

SEO 진단

1-1. <head> 메타 영역

항목현재 상태평가
<title> "모닝브루 — 망원동 핸드드립 카페" ✅ 양호 브랜드+지역+카테고리 포함
<meta description> 없음 ❌ 치명적 검색 결과 설명문 없음
canonical 없음 ❌ 누락 중복 색인 위험
Open Graph 없음 ❌ 누락 SNS 공유 미리보기 없음
Twitter Card 없음 ❌ 누락
lang="ko" 있음 ✅ 정상
viewport 있음 ✅ 정상

1-2. 헤딩 구조 (H1 ~ H3)

태그현재 텍스트평가
H1 "모닝브루" ❌ 취약 브랜드명만, "핸드드립 카페·망원동" 키워드 없음
H2 오늘의 메뉴 / 영업시간 / 오시는 길 / 픽업 예약 ⚠️ 약함 키워드 연관성 낮음
H3 예약 추가 / 예약 목록 ⚠️ 약함 SEO 가치 없는 UI 텍스트

1-3. 콘텐츠 & 키워드

항목현재 상태평가
핵심 키워드 밀도 "핸드드립", "망원동", "스콘", "원두" 언급 ⚠️ 얕음 설명 텍스트 부족
가시 텍스트 분량 약 300자 미만 ❌ 심각 Thin Content 수준
이미지 없음 (이모지·SVG만 사용) ⚠️ 없음 alt 최적화 불가
외부 링크 Instagram, 카카오채널, tel: ✅ 양호 rel="noopener" 적용됨

1-4. 구조화 데이터 (Structured Data)

스키마현재평가
JSON-LD LocalBusiness 없음 ❌ 치명적 로컬 검색 노출 불가
BreadcrumbList 없음 ❌ 누락
Menu Schema 없음 ❌ 누락

1-5. 기술적 SEO

항목현재 상태평가
모바일 반응형 CSS 반응형 적용 ✅ 정상
Google Fonts 로딩 동기 로드 (font-display 미적용) ⚠️ 렌더링 블로킹
sitemap.xml 없음 ❌ 누락
robots.txt 없음 ❌ 누락
NAP 일관성 이름 ✅ / 주소 불완전 / 전화 가번호 ❌ 불완전

Section 02

GEO 진단

GEO(Generative Engine Optimization)란 ChatGPT, Perplexity, Google AI Overview 등 생성형 AI가 답변을 생성할 때 이 페이지를 출처로 인용하거나 정보를 추출할 수 있도록 최적화하는 작업입니다. 기존 SEO와 달리 구조화된 사실 정보·FAQ·엔티티 명확성이 핵심입니다.

2-1. 엔티티 명확성 — "모닝브루는 어떤 곳인가?"

항목현재 상태AI 추출 가능?
상호명 모닝브루 (전체 일관) ✅ 가능
업종 title에만 "핸드드립 카페" ⚠️ 본문 반복 필요
위치 "서울 망원동 골목 끝" (도로명 없음) ❌ 주소 부정확
영업시간 화~일 08:00–20:00 명시 ✅ 가능
전화번호 02-123-4567 (가번호) ❌ 실번호 필요
메뉴·가격 4가지 + 가격 명시 ✅ 가능
고유 강점 "매일 구운 스콘", "핸드드립" ⚠️ 설명 너무 짧음

2-2. 구조화 데이터 — AI 신뢰도

스키마현재필요성
