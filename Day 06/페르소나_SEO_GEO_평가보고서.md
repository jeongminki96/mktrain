## 2. 페르소나별 평가 상세

### 2-1. 구글봇 (Google Crawler)
**점수: 74 / 100 · C+**

**잘 된 점**
- JSON-LD CafeOrCoffeeShop 스키마가 Menu·MenuItem·OpeningHoursSpecification·GeoCoordinates까지 포함, 구조화 데이터 완성도 높음
- FAQPage 스키마를 Microdata 방식으로도 추가 구현, Rich Result 트리거 경로가 두 개로 확보
- robots.txt에 Sitemap URL이 명시, sitemap.xml이 유효한 XML 형식으로 존재

**문제점**
- 이미지 없음 — `<img>` 태그 0개, alt 텍스트 전무, 비주얼 콘텐츠 신호 부재
- JSON-LD streetAddress가 "망원동 골목 끝", postalCode·도로명 주소 누락 → NAP 불완전으로 로컬 신뢰도 하락
- og:image · twitter:image 미존재 → SNS 공유 썸네일 없음, Google Discover 사실상 제외
- H1이 "모닝브루" 단독, "핸드드립 카페·망원동" 키워드 포함 없음
- sitemap.xml lastmod가 2026-06-23(미래 날짜) → 크롤러 신뢰도 저하 요인
- 예약 폼 섹션이 JS 의존 동적 콘텐츠 → 초기 HTML 파싱 시 실질 콘텐츠로 미인식

**누락 항목**
- og:image / twitter:image
- 실제 `<img>` 이미지 및 alt 텍스트
- 완전한 도로명 주소 (postalCode 포함)
- favicon
- BreadcrumbList 스키마
- 실 전화번호 및 실 주소

### 2-2. 생성형 AI 엔진 (ChatGPT / Perplexity)
**점수: 74 / 100 · B**

**이 페이지로 답할 수 있는 질문**
- "모닝브루 영업시간이 어떻게 돼요?" → JSON-LD + FAQPage + 본문 삼중 일치, 고신뢰 답변 가능
- "모닝브루 픽업 예약 방법 알려줘" → FAQPage itemprop="acceptedAnswer" 직접 인용 가능
- "망원동 핸드드립 커피 가격" → MenuItem + Offer(price·priceCurrency) 4종 파싱 가능

**답할 수 없는 질문**
- "모닝브루 정확한 주소 알려줘" → streetAddress가 자연어 서술, 도로명·우편번호 없음
- "모닝브루 주차 가능한가요?" → amenityFeature 정보 전무
- "오늘 모닝브루에 자리 있나요?" → 좌석 수·예약 필요 여부 없음
- "모닝브루 와이파이 돼요?" → 편의시설 정보 전무

**잘 된 점**
- JSON-LD CafeOrCoffeeShop 스키마 완성도 상위 수준 (영업시간·메뉴·가격·좌표·sameAs 전부)
- FAQPage 5개 Q&A 실용적 질문으로 구성, 직접 인용 가능
- 텍스트-스키마 일관성 완전 일치 (모순 없음)

**문제점**
- 실 도로명 주소 부재 → 위치 기반 추천에서 탈락 위험
- 전화번호가 가상 번호(02-123-4567) → AI 인용 시 사용자 연락 실패
- JSON-LD에 `@id` 없음 → Knowledge Graph 연결 약화
- og:image 누락 → AI 요약 카드 비주얼 없음

### 2-3. 타깃 고객 — 박지수 (32세 직장인)
**점수: 72 / 100 · B**

**검색 여정 단계별 경험**
- **검색 단계:** title + description에서 "미리 주문하고 바로 픽업" 확인 → 클릭 의지 생김
- **클릭 단계:** Hero에 픽업 메시지는 보이지만, 영업시간이 첫 화면에 없어 "지금 열었나?" 5초 안에 확인 불가
- **페이지 내 단계:** 예약 폼이 4번째 섹션, 스크롤을 많이 내려야 도달. 예약 후 카페에 실제 전달됐는지 불분명

**잘 된 점 (사용자 목소리)**
- "미리 주문, 도착하면 바로 픽업" 문구가 Hero 정중앙에 있어 내 상황과 딱 맞는 카페임을 첫눈에 알았다
- 영업시간이 명확하게 정리돼 있어 스크롤하면 바로 확인된다
- FAQ에 "오늘의 원두는 어디서 확인하나요?" 답변이 있어 스콘 매진 허탕을 막는 방법을 알 수 있었다

**불편한 점 (사용자 목소리)**
- Hero에 영업시간이 없다. "지금 열었나?" 확인이 안 돼서 두 번 스크롤해야 한다
- 픽업 예약 폼이 페이지 맨 하단 4번째 섹션, 메뉴·오시는 길·CTA 세 섹션을 지나야 한다
- 예약 추가 후 카페 측에 실제 전달됐는지 알 수 없다 (로컬스토리지에만 저장)
- "오시는 길"에 지도 링크가 없다. 골목 안이라고만 써있으면 처음 가는 카페는 헤맨다
- 스콘 재고 현황이 실시간으로 안 보인다. 페이지 안에 재고 상태 배지 하나만 있어도 됐을 텐데

### 2-4. SEO 전문 컨설턴트 (10년 경력)
**점수: 71 / 100 · B**

| 영역 | 점수 | 핵심 코멘트 |
|------|------|------------|
| 기술 SEO | 20 / 25 | JSON-LD·OG·Twitter Card 완비, og:image 누락 + favicon 없음 |
| 온페이지 SEO | 17 / 25 | title·description 우수, H2 5개 중 단 하나도 "망원동 카페" 핵심 키워드 없음, 본문 약 600자로 권장(1,000자+) 미달 |
| 로컬 SEO | 16 / 25 | GeoCoordinates 존재, 실 도로명 주소·postalCode 없음, GMB 연동 전무, sameAs에 네이버 플레이스 URL 부재 |
| 콘텐츠 SEO | 18 / 25 | FAQ 품질 양호, E-E-A-T 신호 가볍게 존재, AggregateRating 스키마 0건 |

**잘 된 점 (전략적)**
- JSON-LD + FAQPage 이중 스키마 구현은 중소 카페 홈페이지 기준 상위 10% 수준
- meta description에 "망원역 2번 출구 도보 6분" 위치 행동 유발 키워드 포함, SERP CTR 유리
- FAQ 5개 질문이 실제 검색 인텐트를 커버, Google 리치 결과(FAQ rich result) 노출 가능성 실재

**전략적 문제점**
- og:image 완전 부재 → SNS CTR 최대 3~5배 손실, Google Discover 사실상 제외
- 실 도로명 주소 없음 → NAP 일관성 0, 로컬 팩 진입 불가
- 본문 텍스트 약 600자 → 경쟁 상위 페이지(1,500~3,000자) 대비 키워드 관련성 열위
- AggregateRating 없음 → 스타 리치 결과 미노출, SERP CTR 최대 25~35% 손실
- 네이버 플레이스 sameAs URL 부재 → 국내 검색 점유율 1위 채널 단절
- sitemap.xml lastmod 미래 날짜 → GSC 색인 우선순위 불이익

**당장 1순위 액션**
- JSON-LD streetAddress에 실제 도로명 주소 + postalCode 입력 → 네이버 플레이스 등록 → sameAs 추가 (로컬 SEO ROI 최고)

### 2-5. 네이버 로컬 검색
**점수: 52 / 100 · C**

**네이버 최적화 신호**
- 주소 데이터 불완전: "망원동 골목 끝"은 플레이스 색인 불가 수준
- 전화번호 가상 번호(02-123-4567): 네이버 플레이스 업체 검증 실패
- 네이버 플레이스 URL sameAs 없음: 지역 검색 핵심 경로 단절

**카카오 생태계 신호**
- 카카오채널 연동 구현됨: sameAs + CTA 버튼 존재 → 카카오맵 신뢰 신호 가능
- 카카오맵 임베드 없음: "오시는 길" 섹션에 지도 임베드 또는 링크 미존재

**잘 된 점**
- 한국어 키워드("망원동·마포구·핸드드립 카페·망원역·픽업 예약") 자연스럽게 반복 포함
- JSON-LD LocalBusiness + FAQPage 이중 스키마, 네이버도 구조화 데이터 참고
- 인스타그램 운영 루틴 명시, 블로그·카페 리뷰 연결 고리 형성

**문제점**
- 네이버 플레이스 미연동: 한국 카페 검색 80% 이상이 이 경로, 홈페이지만 있으면 지역 검색 노출 제로
- 실주소 부재: 네이버 지도·카카오맵 핀 생성 불가
- 네이버 블로그·카페 유입 콘텐츠 없음: VIEW 탭 노출 불가
- og:image 없음: 네이버 카페·블로그 URL 공유 시 썸네일 없음
- canonical URL이 GitHub Pages → 실제 도메인 없이 신뢰도 하락
